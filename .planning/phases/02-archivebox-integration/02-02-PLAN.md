Run the archive job with ACT and inspect the output.

**Pre-run cleanup** (ensure fresh artifact output):
```bash
rm -rf /tmp/act-artifacts
```

**Run the archive job:**
```bash
cd /Users/welch/Code/cve/refgha
act workflow_dispatch \
  -j archive \
  -W .github/workflows/archive-cve.yml \
  --input cve_id=CVE-2021-44228
```

NOTE: The `--input` flag is required because the workflow has `workflow_dispatch` inputs declared. Even though the archive job doesn't use `inputs.cve_id` (it has a hardcoded URL), ACT may require all declared inputs to be present.

**Expected behavior during run:**
- ACT pulls `ghcr.io/catthehacker/ubuntu:act-24.04` on first run (may take several minutes)
- Docker pulls `archivebox/archivebox` on first run (may take 1-2 minutes)
- ArchiveBox prints logs including "Added 1 URLs" and extractor results
- The run ends with exit 0

**After the run, inspect artifacts:**
```bash
find /tmp/act-artifacts -type f | sort
```

Expected output should include files matching:
- `**/pdf.pdf`
- `**/screenshot.png`
- `**/warc.tgz`

**If the WARC is missing:** Check whether the `warc/` directory appeared in the snapshot dir. If WARC is missing but wget ran successfully, the issue is likely `SAVE_WARC=False` being overridden somewhere. Check the step log output.

**If pdf.pdf is missing:** The most likely cause is `SAVE_PDF=True` not being passed correctly (YAML indentation issue). Inspect the docker run command in the step logs.

**Collect evidence for the checkpoint:**
```bash
# List artifact files
find /tmp/act-artifacts -type f | sort

# Confirm all three file types present
find /tmp/act-artifacts -name "pdf.pdf"      | grep -q . && echo "pdf.pdf OK"      || echo "MISSING: pdf.pdf"
find /tmp/act-artifacts -name "screenshot.png" | grep -q . && echo "screenshot OK" || echo "MISSING: screenshot.png"
find /tmp/act-artifacts -name "warc.tgz"     | grep -q . && echo "warc.tgz OK"     || echo "MISSING: warc.tgz"
```

Record the output in the checkpoint below.

   ```bash
   file $(find /tmp/act-artifacts -name "warc.tgz")
   ```
   Expected: `gzip compressed data`
